# 现场使用说明

## 1. 基础设置

1. 打开“设置”页面。
2. 连接类型选择 `内置 SFTP` 或 `WinSCP`。
3. 填写服务器地址、端口、用户名、密码、服务器目录。
4. 服务器文件保持为 `slaverMB_1.map`。
5. 第三方工具只填写安装目录，不填写可执行文件路径。
6. 规则导入只在现场规则变更时使用。

## 2. 本地替换

1. 进入“本地替换”页面。
2. 选择当前风场。
3. 选择本机故障 MAP 和目标正常 MAP，或将待处理文件放入 `input_maps`。
4. 添加仿真关系，例如 `1 仿 2`、`3 仿 6`。
5. 点击“本地批量替换”。
6. 查看底部日志确认处理结果。

输出文件：

- `output_maps`：仿真后的 MAP 文件。
- `update/slaverMB_1.map`：待上传服务器的文件。
- `reports`：替换报告。

## 3. 云端替换

### 下载服务器 MAP

“下载服务器 MAP”只执行下载动作：

- 从服务器目录下载 `slaverMB_1.map`。
- 保存到 `download/slaverMB_1.map`。
- 文件名和文件内容不做修改。

### 一键仿真

“一键仿真”按以下顺序执行：

1. 备份服务器原始 `slaverMB_1.map`。
2. 下载服务器原始 `slaverMB_1.map` 到 `download`。
3. 本地备份原始文件到 `backup/slaverMB_1.map`。
4. 按当前风场规则和仿真关系生成替换结果。
5. 写入 `update/slaverMB_1.map`。
6. 上传覆盖服务器原始 `slaverMB_1.map`。

### 上传 update

“上传 update”按以下顺序执行：

1. 备份服务器原始 `slaverMB_1.map`。
2. 上传 `update/slaverMB_1.map`。
3. 覆盖服务器目录中的 `slaverMB_1.map`。

### 恢复最近备份

“恢复最近备份”从本地 `backup` 中选取最近的 `.map` 文件并上传到服务器，服务器端文件名仍固定为 `slaverMB_1.map`。

## 4. 服务器备份命名

服务器备份文件不能与 `slaverMB_1.map` 同名。

命名规则：

- 单条关系：`1-2-before.map`
- 多条关系：`1-2_3-6-before.map`
- 无有效关系：`manual-before.map`

风机编号从风机名最后一段数字提取：

- `F1-01FJ` -> `1`
- `F1-02FJ` -> `2`
- `F3-06FJ` -> `6`

## 5. 日志排查

本地替换和云端替换页面底部均显示运行日志。

常见检查项：

- 连接是否成功。
- 下载路径是否为服务器原始 `slaverMB_1.map`。
- 上传目标是否为服务器完整路径下的 `slaverMB_1.map`。
- 服务器备份名是否符合 `1-2-before.map` 或 `1-2_3-6-before.map`。
- `reports` 中是否生成替换报告。
